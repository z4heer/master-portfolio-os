### Part 2: Step-by-Step Junior Developer Implementation Guide

If you (or a junior teammate) were to start this manually today, follow these steps to ensure "Solution Architect" quality:

#### Step 1: Define the Data Interface
Don't start coding logic until the "Contract" with the backend is set.
*   **File:** `src/app/core/models/product.model.ts`
*   **Task:** Match the backend JSON exactly.
```typescript
export interface Product {
  id: string; // UUID from PostgreSQL
  name: string;
  description: string;
  price: number;
  category: string;
  is_active: boolean;
  stock_quantity?: number; // From the Inventory relationship
}
```

#### Step 2: Build the API Bridge (Service Layer)
*   **File:** `src/app/features/catalog/services/product.service.ts`
*   **Instruction:** Use `HttpParams` for the search and category filters you verified in Sprint 3.
*   **Junior Tip:** Remind the developer to use `catchError` to handle cases where the FastAPI backend might be down.

#### Step 3: Create the Responsive UI
*   **File:** `src/app/features/catalog/product-list/product-list.component.html`
*   **Component:** Use **Angular Material Grid** or **Flex-Layout**.
*   **Verification:** Ensure that if a product is "Out of Stock" (verified via your `inventory` table checks), the "Add to Cart" button is disabled.

#### Step 4: Routing & Navigation
*   **File:** `app.routes.ts`
*   **Task:** Map the catalog as the default route.
```typescript
{ path: 'products', component: ProductListComponent },
{ path: 'products/:id', component: ProductDetailComponent },
{ path: '', redirectTo: 'products', pathMatch: 'full' }
```
