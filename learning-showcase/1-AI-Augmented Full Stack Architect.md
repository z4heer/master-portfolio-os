# AI PROJECT EXECUTION MODEL

## Human Does

| Human Responsibility   | Why                       |
| ---------------------- | ------------------------- |
| Business decisions     | AI lacks ownership        |
| Requirement validation | Avoid wrong assumptions   |
| Code execution         | AI cannot run infra fully |
| Final review           | Quality control           |
| Production deployment  | Safety                    |
| Security approvals     | Critical                  |
| UAT                    | Real business testing     |

## AI Does

| AI Responsibility       | Example       |
| ----------------------- | ------------- |
| Generate drafts         | APIs          |
| Suggest architecture    | Microservices |
| Create test cases       | CRUD tests    |
| Generate Docker configs | DevOps        |
| Debug errors            | Stack traces  |
| Write documentation     | README        |
| Create SQL schema       | DB design     |

# GOLDEN RULE

Best results happen when:

```text id="l5f4x0"
Human manages project.
AI accelerates execution.
```

Not:

```text id="v14iqt"
AI builds everything automatically.
```

That mindset gives much higher quality and maintainability.
