Project Rules

1. Follow Clean Architecture.

2. Follow Repository-Service pattern.

3. Repository contains persistence only.

4. Business logic belongs only in Service Layer.

5. No shortcut implementations.

6. No temporary hacks.

7. No breaking existing architecture.

8. Maintain modular boundaries.

9. Follow existing naming conventions.

10. Code must be production-ready.

11. Preserve backward compatibility unless explicitly approved.

12. Every implementation should be testable.

13. Reuse existing components before creating new ones.

14. Prefer maintainability over cleverness.

15. Respect current project structure.
